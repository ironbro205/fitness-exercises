# 헬스앱 "AI 루틴 구성" 근성장 개선 계획서

## 1) 한 줄 요약
지금 AI는 "한 세션"을 그럴듯하게 잘 짜지만, **근성장의 1순위 동력인 "주간 볼륨(부위별 주당 세트 수)"을 닫힌 고리로 관리하지 못하고, 사용자가 코치에게 말한 부상·선호를 루틴에 반영하지 못한다.** 볼륨 폐루프·RIR(세트에서 남긴 반복 수)·부상 회피 3가지만 채워도 근거 기반 처방 품질이 크게 오른다.

## 2) 현재 AI가 하는 것 & 약점
현재: 종목 풀에서만 5~7개를 골라(복합/고립·신장강조·1RM 태그 부착) 부족 부위·다양성·순서·신규종목 %1RM을 규칙으로 처방. 볼륨 근거는 프롬프트에 텍스트로만 주입.

핵심 약점(감사 기반):
- **부상·선호 누락(가장 큼)**: "어깨 아파"/"데드 싫어"를 코치에게 말해도 `generateFullRoutine`은 `COACH_MEMORY`를 안 읽어 회피가 안 됨(ai.js:921 경로).
- **주간 볼륨 폐루프 없음**: 세션을 매번 따로 생성 → "이 부위 이번 주 몇 세트 남았나(목표 10~20까지)"를 계산해 세션 크기를 맞추지 못함.
- **진단 맹점**: 최근 2주 0세트 부위는 부족 목록에서 아예 빠지고(domain.js:779), 4~10세트(최적 하한 미달)도 경고가 안 뜸 → "가장 안 한 부위가 안 보임".
- **RIR·반복범위가 근비대에 덜 맞음**: 평상시·도전 모드의 목표 RIR 규칙이 없고(회복 모드만, ai.js:1040), 메인 반복을 근력 편향 "5-8"로 고정(ai.js:1034).
- **결정론적 계산 미사용**: `getProgressiveRecommendation`(코드가 정확히 +2.5kg 산출)을 세션 UI만 쓰고 루틴 AI엔 원문만 줘 눈대중 판단, 게다가 표준(2세션 연속)이 아닌 1세션 만에 증량.
- **휴식·디로드 미출력**: 응답 스키마에 `rest` 필드가 없어 휴식 미처방(ai.js:1057), 디로드 주차가 규칙에 연결 안 됨.

## 3) 개선안 (우선순위순)

### ① 코치 기억(부상·제약·선호)을 루틴 생성에 주입  [영향 高 / 노력 低]
- **무엇을**: `generateFullRoutine` 컨텍스트에 `COACH_MEMORY`(부상/제약/선호/목표)를 추가하고, "제약·부상 부위 종목은 회피/대체, 싫어하는 종목 제외" 규칙 1줄 추가.
- **왜**: 안전이 최우선이며(KB 부상 트리아지: 날카로운 관절통=중단·대체), 지금은 코치 대화에만 반영돼 루틴이 무시함 — 감사가 "가장 큰 누락"으로 지목.
- **앱 반영법**: `buildUserContext` 결과 뒤에 `parseCoachMemoryBlock` 텍스트를 붙이고, 규칙에 "'제약' 목록의 부위·동작은 통증 없는 종목으로 대체(예: 어깨 임핀지 → 팔꿈치 활짝 프레스 대신 중립 그립)" 삽입.

### ② 주간 볼륨 폐루프 + 진단 맹점 수정  [영향 高 / 노력 中]
- **무엇을**: 부위별 "최근 주간 세트 → 목표(기본 12~16, 범위 10~20)까지 남은 세트"를 코드가 계산해 컨텍스트로 주고, AI가 이번 세션 세트 수를 그 격차에 맞추게. 0세트 부위도 부족 목록에 넣고, 4~10세트엔 "🟡 하한 미달" 등급 신설.
- **왜**: 주간 볼륨이 근비대 1차 동력이고 12~20세트가 실용 최적(Pelland 2024; Baz-Valle 2022; Schoenfeld·Ogborn·Krieger 2017). 단, **20을 생리적 상한처럼 하드컷 금지** — 초과분은 "수확 체감(효과 줄지만 이득은 지속)"으로만 안내(verify: 20+도 일부는 성장). 유연 분할이라 세션 단발 생성이 특히 취약.
- **앱 반영법**: domain.js:779의 `if(weeklyVol>0)`를 완화해 0세트도 표시, domain.js:783 구간(4~10)에 🟡 등급 추가. 컨텍스트에 "가슴 남은 세트: 목표14 − 현재6 = 8세트 필요" 식 표. **한 세션 한 부위 직접 세트 ≤ 약 8**(소프트 상한 — verify: 세션당 상한 근거는 잠정치, 데이터 희박)로 두고 초과분은 "주 2회로 분할" 안내.

### ③ 모든 세트 RIR 밴드 명시 + 반복범위 근비대로 재조정  [영향 高 / 노력 低]
- **무엇을**: 회복 모드뿐 아니라 모든 세트에 RIR 밴드 처방. 메인 반복을 근력 편향 "5-8"에서 근비대 중심 "6-10"으로.
- **왜**: 실패 근접도가 근비대에 특이적으로 작용(Robinson 2024; Refalo 2023) — 실패에 가까울수록 소폭 유리하나 완전 실패(0 RIR)는 불필요. 5~30회가 실패 근접 시 동등하되 6~12회가 효율 스윗스팟(Schoenfeld·Grgic 2021). 지금은 예시 `rir:2`에만 의존.
- **앱 반영법**: 규칙에 "복합=RIR 2~3, 고립/머신=RIR 0~2(마지막 세트 0~1), 무거운 바벨 복합에 완전 실패 넣지 말 것" 추가. 반복: 메인 복합 6~10, 보조 복합 8~12, 고립 10~20. **주의: RIR을 하드 게이트로 쓰지 말 것**(verify: 자가보고 RIR 부정확, 4 RIR을 "정크"로 낙인 금지) — "밴드+마지막 세트 RIR로 마감 판단"의 소프트 넛지로.

### ④ 신규종목 %1RM↔반복 정합 + 결정론적 진행 엔진 연결  [영향 中 / 노력 中]
- **무엇을**: 신규 종목 무게를 반복 목표에 맞게(8회 목표면 70%가 아니라 ~77%), 그리고 `getProgressiveRecommendation`(코드 계산값)을 컨텍스트에 직접 주입해 AI 눈대중 대신 사용. 증량 트리거를 표준인 "2세션 연속 상단"으로.
- **왜**: 지금은 메인 70% 1RM(≈12RM)로 5-8회를 시키는 부하-반복 불일치(감사 지목). 더블 프로그레션(무게 대신 반복부터 올리기)은 표준이며(Plotkin 2022 — 단, 8주 단일 RCT라 과신 금지), 코드값과 AI 판단이 어긋나면 일관성이 깨짐.
- **앱 반영법**: 반복→%1RM 근사표 제공(6회≈82%, 8회≈77%, 10회≈73%, 12회≈68%, 15회≈65%) — **집단 평균이라 상체 프리웨이트·고립은 편차 큼, 추정치로만 표기하고 RIR로 재보정**(verify). 세션당 종목별 워킹세트 기본 3~4개.

### ⑤ 분할(간접)세트 볼륨 정합 + 필수 고립 슬롯  [영향 中 / 노력 低]
- **무엇을**: secondary(보조근) 가중치를 0.3→0.5로(이미 `secondary` 배열 있음, domain.js:751 한 줄), 그리고 "팔(이두·삼두)·측면/후면 어깨·종아리는 복합운동에서 0.5씩만 쌓이므로 직접 고립 종목 필수" 규칙 추가.
- **왜**: 분할(fractional) 계산이 세 방식 중 예측력 최고(Pelland 2025). **단 0.5는 "3개 관례 중 최적"일 뿐 정밀값 아님(범위 0.3~0.7) — 조정 가능한 휴리스틱으로만.** 결론(팔·측면어깨 직접 고립 필요)은 어느 값이든 유효.
- **앱 반영법**: 복합만으로 짠 세션이면 "팔/측면어깨 볼륨 미달" 경고 후 고립 자동 보충. 후면 어깨는 프레스로 대체 불가 → **어깨 세션에 리어델트 고립 필수 슬롯**(근거 견고: EMG+역학).

### ⑥ 신장강조·부위 커버 소프트 규칙 + 순서=약점 우선 재프레이밍  [영향 中 / 노력 低]
- **무엇을**: 부위별 신장강조(근육이 늘어난 자세에서 부하 큰) 종목 최소 1개 가중, 가슴 상부(인클라인)+중/하부·등 수직+수평 커버. 순서 규칙을 "약점/강조 부위를 맨 앞(신선할 때)"으로.
- **왜**: 신장강조가 삼두(오버헤드)·햄스트링(시티드컬)·종아리(스탠딩)에서 일관되게 유리(Maeo 2021·2023). **단 신장매개 성장 효과는 작고 논쟁적, 가슴 상·중·하와 어깨 3갈래·등 수직/수평은 EMG 기반이라 국소 성장 예측력 약함 — 사용자에게 단정 말고 "저비용 헤지"로만**(verify). 종목 순서는 근비대에 무관(Nunes 2021, 효과크기 0.03) → 순서는 "무엇을 먼저 강하게 할지" 도구.
- **앱 반영법**: 종목에 stretch_bias 태그로 동점 시 소프트 가중(주 지렛대로 과대선전 금지). 순서 규칙을 "강조 부위 1~2번 슬롯"으로. **시간 제한 없음 조건이므로 슈퍼셋·`duration` 상한은 우선순위에서 제외**(불필요한 시간 압박 만들지 말 것).

### ⑦ 디로드·주기화를 "피로 신호" 기반으로 연결  [영향 中 / 노력 中]
- **무엇을**: `cyclePhase=디로드`이거나 정체·컨디션 저하 신호일 때 볼륨 -40~50%·RIR 3~4로 낮추는 규칙 추가(지금은 컨디션 반응형 -20%뿐, cyclePhase는 텍스트로만 들어감).
- **왜**: 디로드는 누적 피로·관절·심리 관리용. **"성장 부스터"로 표기 금지** — 직접 RCT(Coleman 2024)는 디로드 주 추가로 성장이 더 늘지 않았고, "4~6주마다" 근거(Bell 2023)는 전문가 설문 합의(의견)이지 실험이 아님(verify). 고정 자동삽입보다 피로 신호 트리거로.
- **앱 반영법**: 트리거=정체(🔥 3세션 동일무게)+컨디션 저하 동시. 디로드 시 "종목 유지, 세트 -40~50%, RIR 3~5, ~7일" 처방 + "이 감량은 성장이 아니라 회복·부상 예방용" 안내.

## 4) 지금 당장 넣을 quick win 3가지
1. **응답 JSON에 `rest`(휴식) 필드 추가**: 스키마에 `"rest":"120-180"` 추가(ai.js:1057) + 규칙 "복합 2~3분/고립 1~2분". KB엔 이미 있는데 출력만 빠져 휴식 미처방(짧은 휴식은 다음 세트 볼륨을 깎음).
2. **평상시·도전 모드 RIR 밴드 한 문단**: ai.js:1040 회복 모드 아래에 "평상시/도전 = 복합 RIR 2~3, 고립 RIR 0~2(마지막 세트 0~1)" 추가 — 지금은 예시 `rir:2`에만 의존.
3. **볼륨 진단 맹점 2줄 수정**: domain.js:779에서 0세트 부위도 부족 목록에 표시, 783 구간(주 4~10세트)에 🟡 "하한 미달" 등급 추가 → 기존 "부족 부위" 리스트가 완전해져 ①·② 프롬프트 정확도가 같이 오름.
